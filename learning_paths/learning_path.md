# VibeLearn 学习路径规划

## 📅 总览

本学习路径基于市场上成熟的学习方法论（如 Bloom 分类法、Spaced Repetition）设计，旨在帮助学习者循序渐进地掌握 AI 辅助编程技能。

---

## 🎯 学习阶段规划

### 阶段一：基础入门（2-3 周）

**目标**：建立 LLM 基础认知

| 周 | 主题 | 学习内容 | 实践任务 |
|:---|:---|:---|:---|
| 1 | LLM 基础 | Transformer 架构、主流模型对比 | 阅读论文、整理笔记 |
| 2 | API 调用 | OpenAI API、Prompt 基础 | 实现简单聊天机器人 |
| 3 | Prompt 工程 | 指令设计、格式控制 | 完成文本分类任务 |

**能力评估**：
- ✅ 能独立调用 LLM API
- ✅ 能设计有效的 Prompt
- ✅ 能处理简单的文本生成任务

---

### 阶段二：进阶实践（3-4 周）

**目标**：掌握 Agent 架构设计

| 周 | 主题 | 学习内容 | 实践任务 |
|:---|:---|:---|:---|
| 4 | Agent 原理 | 核心组件、工作流程 | 理解架构图 |
| 5 | 代码生成 | AI 写代码、代码验证 | 实现代码生成器 |
| 6 | 任务编排 | 工作流设计、状态管理 | 构建数据处理 Agent |
| 7 | 结果验证 | AI 自检、执行验证 | 完善 Agent 工作流 |

**能力评估**：
- ✅ 能设计 Agent 架构
- ✅ 能实现 AI 代码生成
- ✅ 能构建完整工作流

---

### 阶段三：高级应用（4-6 周）

**目标**：掌握 RAG 和复杂系统

| 周 | 主题 | 学习内容 | 实践任务 |
|:---|:---|:---|:---|
| 8 | RAG 基础 | 检索增强、向量数据库 | 学习向量数据库使用 |
| 9 | RAG 实践 | 文档处理、检索策略 | 实现简单 RAG 系统 |
| 10 | 多模态 | 图像、语音、视频 | 构建多模态 Agent |
| 11 | 多 Agent | 协作模式、任务分配 | 实现多 Agent 系统 |
| 12 | 企业部署 | 性能优化、安全合规 | 完成企业级项目 |

**能力评估**：
- ✅ 能构建 RAG 系统
- ✅ 能实现多模态应用
- ✅ 能部署企业级 AI 应用

## 📊 难度评估体系

### 难度等级定义

| 等级 | 图标 | 描述 |
|:---|:---|:---|
| 1 | ⭐☆☆ | 入门级 - 概念简单、代码量少 |
| 2 | ⭐⭐☆ | 基础级 - 需要理解概念、中等代码量 |
| 3 | ⭐⭐⭐ | 进阶级 - 需要深入理解、复杂代码 |
| 4 | ⭐⭐⭐⭐ | 高级 - 需要创新思维、系统设计 |

### 评估维度

| 维度 | 说明 | 权重 |
|:---|:---|:---|
| **概念理解** | 对核心概念的理解难度 | 30% |
| **代码复杂度** | 实现代码的复杂程度 | 30% |
| **调试难度** | 排查问题的难度 | 25% |
| **创新性** | 需要创新思维的程度 | 15% |

## 🎨 学习方法论

### 1. 主动学习法
- 边学边练，每学习一个概念就动手实践
- 尝试解释给他人听，检验理解程度

### 2. 项目驱动
- 以项目为核心，边做边学
- 从简单到复杂，逐步积累

### 3. 间隔重复
- 定期复习已学内容
- 建立知识体系图

### 4. 反馈闭环
- 收集反馈，持续改进
- 记录学习日志
