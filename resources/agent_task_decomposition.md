## 核心原则

Agent 任务拆解是 AI Agent 成功的基础，遵循以下核心原则：

1. **可执行性（Executability）：每个子任务必须是 Agent 可以实际执行的
2. **明确性（Clarity）：每个子任务都有清晰的输入和输出
3. **独立性（Independence）：子任务之间尽可能独立，减少耦合
4. **可验证性（Verifiability）：每个子任务的结果可以验证
5. **可组合性（Composability）：子任务可以组合成更大的任务

## 标准任务拆解六步法

### 第 1 步：理解任务目标

🔍 **动作：彻底理解用户的最终需求和期望结果

**关键问题清单：**
- 用户想要解决什么问题？
- 期望的输出格式是什么？
- 有哪些约束条件（时间、成本、质量）？
- 成功的衡量标准是什么？

**Prompt 示例：
```
请理解以下任务的目标：
{用户任务描述}

请回答：
1. 核心问题是什么？
2. 期望的输出格式是什么？
3. 有哪些约束条件？
```

---

### 第 2 步：分析任务分解成主要阶段

📦 **动作：将大任务分解为多个主要阶段

**常用分解维度：**
- 按时间顺序（输入→处理→输出
- 按逻辑依赖关系
- 按功能模块
- 按数据流程

**示例框架：
```
阶段1：数据收集
阶段2：数据处理
阶段3：分析计算
阶段4：结果生成
阶段5：验证优化
```

---

### 第 3 步：识别所需资源和工具

🛠️ **动作：确定每个阶段需要的资源和工具

**工具类型：**
- API 调用
- 文件操作
- 代码执行
- 外部服务
- 内部知识库

**表格模板：**

| 阶段 | 需要的工具/资源 | 用途 |
|------|--------------|------|
| 阶段1 | 地址转经纬度 API | 将地址转换 |
| 阶段2 | Python 代码执行 | 计算距离 |

---

### 第 4 步：设计子任务输入输出

📝 **动作：为每个子任务明确输入和输出

**规范：**
- 明确的输入格式
- 期望的输出格式
- 数据传递方式
- 错误处理

---

### 第 5 步：构建执行流程

🔄 **动作：确定子任务的执行顺序和依赖关系

**流程图示例：**
```
开始
   ↓
[子任务1] → [子任务2] → [子任务3]
   ↓           ↓           ↓
[验证1]    [验证2]    [验证3]
   ↓           ↓           ↓
   └───────────┴───────────┘
                ↓
           [最终验证]
                ↓
              结束
```

---

### 第 6 步：验证和优化

✅ **动作：验证任务拆解的合理性

**验证要点：**
- 每个子任务是否必要吗？
- 顺序合理吗？
- 有遗漏吗？
- 有冗余吗？

### 案例：地址周边站点筛选

基于项目中的 [agent_workflow.py](file:///workspace/practice/level2_intermediate/agent_workflow.py) 为例：

**用户任务：根据地址筛选周边 3 公里内的站点

| 步骤 | 子任务 | 输入 | 输出 | 工具 |
|------|--------|------|------|------|

## 实践示例

让我们用另一个例子：**自动数据分析报告

**用户任务：**
> "帮我分析销售数据，生成分析报告

**任务拆解：**

**第 1 步：理解任务目标**
- 核心问题：分析销售数据，找出趋势
- 期望输出：分析报告（文字 + 图表
- 约束：数据在 sales.csv，报告需要包含趋势、异常值、建议

**第 2 步：分解成主要阶段**
```
阶段1：数据加载和探索
阶段2：数据清洗
阶段3：趋势分析
阶段4：可视化生成
阶段5：报告撰写

**第 3 步：识别工具**
- Pandas（数据处理）
- Matplotlib（可视化）
- LLM（报告生成）

**第 4 步：设计输入输出**
| 子任务 | 输入 | 输出 |
|--------|------|------|
| 数据探索 | sales.csv | 数据概览 |
| 数据清洗 | 原始数据 | 清洗后数据 |
| 趋势分析 | 清洗数据 | 分析结果 |
| 可视化 | 分析结果 | 图表 |
| 报告 | 图表+分析 | 最终报告 |

**第 5 步：构建流程**
```
开始
  ↓
[数据加载]
  ↓
[数据探索]
  ↓
[数据清洗]
  ↓
[趋势分析]
  ↓
[可视化]
  ↓
[报告生成]
  ↓
结束
```

**第 6 步：验证优化**
- ✅ 每个步骤必要
- ✅ 顺序合理
- ✅ 无遗漏无冗余

## 最佳实践

### 1. 使用结构化 Prompt

给 Agent 的任务拆解 Prompt 模板：

```
你是一个任务拆解专家。请将以下任务拆解为可执行的子任务。

任务：{任务描述}

请按照以下步骤进行：

1. 理解任务目标
2. 分解为 3-6 个主要阶段
3. 确定每个阶段的输入输出
4. 确定所需工具
5. 构建执行流程
6. 验证合理性

请用 JSON 格式返回：
{
  "goal": "任务目标",
  "stages": [
    {
      "name": "阶段名称",
      "description": "阶段描述",
      "input": "输入",
      "output": "输出",
      "tools": ["工具1", "工具2"]
    }
  ],
  "workflow": "执行流程描述",
  "validation": "验证结果"
}
```

### 2. 迭代优化

- 先从简单任务开始练习
- 逐步增加复杂度
- 记录成功和失败的案例
- 持续优化拆解方法

### 3. 错误处理

每个子任务都应该考虑：
- 如果失败了怎么办？
- 需要重试吗？
- 如何通知用户？
- 有备选方案吗？

## 总结

Agent 任务拆解是一个需要练习的技能，通过：

1. **理解目标
2. **分解阶段
3. **识别资源
4. **设计 IO
5. **构建流程
6. **验证优化

这六个步骤，你可以构建出可靠、可执行的 Agent 工作流。

记住：**清晰的任务拆解是 Agent 成功的一半！
