## LLM 基础原理

### 什么是大语言模型（LLM）

大语言模型是基于Transformer架构的深度学习模型，能够理解和生成人类语言。

### 核心概念

#### 1. Transformer 架构
- 自注意力机制（Self-Attention）
- 多头注意力（Multi-Head Attention）
- 位置编码（Positional Encoding）

#### 2. 训练过程
- **预训练（Pre-training）**：在海量文本上学习通用知识
- **微调（Fine-tuning）**：在特定任务上进行调整
- **提示学习（Prompt Learning）**：通过提示词引导模型行为

#### 3. 关键能力
- 文本生成
- 理解推理
- 代码生成
- 多模态理解

### 主流模型对比

| 模型 | 发布时间 | 特点 |
|:---|:---|:---|
| GPT-3 | 2020 | 首次展示大模型能力 |
| GPT-4 | 2023 | 多模态能力、更强推理 |
| DeepSeek-V3 | 2024 | 开源、长上下文、代码能力强 |
| Qwen | 2023 | 阿里开源、中文支持好 |

## Agent 架构原理

### Agent 定义

AI Agent 是一种能够感知环境、做出决策、执行动作的智能系统。

### 核心组件

#### 1. Prompt 模块
- 提示词设计
- 上下文管理
- 指令工程

#### 2. LLM 模块
- 模型选择
- API 调用
- 响应解析

#### 3. 工作流模块
- 任务拆解
- 步骤编排
- 状态管理

#### 4. 工具调用模块
- 代码执行
- API 调用
- 数据库查询

### Agent 架构模式

#### 1. 单 Agent 模式
```
用户输入 → Prompt → LLM → 工具执行 → 结果
```

#### 2. 多 Agent 协作模式
```
用户输入 → 任务分配 → 多个 Agent 协作 → 汇总结果
```

### 关键技术

#### 1. 任务拆解
- 递归拆解
- 优先级排序
- 依赖分析

#### 2. 结果验证
- AI 自检
- 执行验证
- 人工审核

#### 3. 反馈机制
- 错误修正
- 学习优化
- 持续改进

### 设计原则
1. **可控性**：确保结果可预测
2. **可验证**：结果可以被检验
3. **可追溯**：过程可以被追踪
4. **鲁棒性**：处理异常情况

## RAG 技术体系

### RAG 定义

RAG（Retrieval-Augmented Generation）是一种结合检索和生成的技术，让 LLM 能够基于外部知识库生成回答。

### RAG 工作流程

```
用户提问 → 检索相关文档 → 构建提示词 → LLM 生成回答
```

### 核心组件

#### 1. 文档处理
- 文档加载
- 文本分割
- 向量化

#### 2. 向量数据库
- 存储向量嵌入
- 相似度检索
- 增量更新

#### 3. 检索策略
- 语义检索
- 混合检索
- 多模态检索

#### 4. 提示词构建
- 上下文注入
- 指令设计
- 格式控制

### 主流向量数据库

| 数据库 | 特点 | 适用场景 |
|:---|:---|:---|
| Pinecone | 托管服务、易用 | 快速原型 |
| Chroma | 轻量、开源 | 本地开发 |
| Milvus | 高性能、可扩展 | 企业级 |
| FAISS | 高效、Facebook | 研究场景 |

### RAG 优化策略
1. **Chunk 优化**：合理的文本分割大小
2. **检索增强**：多级检索、重排序
3. **提示词优化**：上下文管理、指令工程
4. **评估体系**：BLEU、ROUGE、人工评估

### 应用场景
- 知识库问答
- 文档摘要
- 代码检索
- 多模态检索

## 核心概念详解

### RAG（Retrieval-Augmented Generation）

**什么是 RAG？**
RAG = **检索增强生成**，是一种结合了信息检索和文本生成的技术，让大语言模型（LLM）能够基于外部知识库来回答问题。

**RAG 的核心组件**

| 组件 | 说明 |
|:---|:---|
| **文档处理** | 将文档切分成块（Chunks） |
| **向量化** | 将文本转换为向量嵌入（Embedding） |
| **向量数据库** | 存储向量，支持相似度检索 |
| **检索策略** | 语义检索、混合检索等 |
| **提示词构建** | 将检索到的上下文注入提示词 |

**RAG 的优势**
- ✅ 减少幻觉（Hallucination）
- ✅ 支持最新信息
- ✅ 可追溯信息来源
- ✅ 保护知识产权

### Function Call（函数调用）

**什么是 Function Call？**
**Function Call** 是 LLM 调用外部工具/函数的能力，让 AI 不是只说话，而是能「做事」。

**Function Call 的工作流程**
```
用户请求 → LLM 分析需求 → 判断需要调用工具 → 调用 Function → 获取结果 → 整合回答
```

**Function Call 的典型应用**
| 应用场景 | 示例 |
|:---|:---|
| **获取实时数据** | 天气查询、股票行情 |
| **外部 API 调用** | 地图、快递、支付 |
| **代码执行** | 运行 Python 代码、SQL 查询 |
| **系统操作** | 文件读写、数据库操作 |

**Function Call 示例**
```python
functions = [
    {
        "name": "get_weather",
        "description": "获取指定城市的天气",
        "parameters": {
            "type": "object",
            "properties": {
                "city": {"type": "string", "description": "城市名"},
            },
            "required": ["city"]
        }
    }
]
```

### MCP（Model Context Protocol）

**什么是 MCP？**
**MCP** = **模型上下文协议**，是一套让 AI 能够与外部资源（文件系统、数据库、API）交互的标准化协议。

**MCP 的核心理念**
- **标准化接口**：统一的工具调用方式
- **可扩展**：易于添加新的工具和数据源
- **安全隔离**：沙箱环境，安全可控
- **双向通信**：AI 和工具之间可以持续交互

**MCP 的典型组件**
| 组件 | 说明 |
|:---|:---|
| **Servers** | 提供工具和资源的服务端 |
| **Tools** | 可调用的具体功能 |
| **Resources** | AI 可访问的数据源 |
| **Prompts** | 预定义的提示词模板 |

---

### Skill（技能）

**什么是 Skill？**
**Skill** 是指 AI 能够完成的特定任务能力，通常是将多个 Function 或工作流组合封装而成。

**Skill 的特点**
- 🎯 **面向任务**：解决具体问题
- 🔧 **可组合**：多个 Skill 可以组合使用
- 📦 **可复用**：一次编写，多次使用
- 🤝 **与人协作**：可以和人类协作完成任务

**Skill 示例**
| Skill 名称 | 功能描述 |
|:---|:---|
| **写代码** | 生成、调试、运行代码 |
| **数据分析** | 读取数据、分析、可视化 |
| **文档处理** | 阅读、总结、翻译文档 |
| **智能助手** | 日程管理、提醒、搜索 |

## 概念对比

### 从「能力层级」看

```
┌─────────────────────────────────────────────────────────────┐
│                         Skill（技能）                         │
│         （面向任务，组合多个工具完成复杂工作）                 │
└─────────────────────────────────────────────────────────────┘
                              ↑
┌─────────────────────────────────────────────────────────────┐
│    Function Call / MCP（工具调用）                           │
│         （调用单个工具，获取外部信息或执行操作）               │
└─────────────────────────────────────────────────────────────┘
                              ↑
┌─────────────────────────────────────────────────────────────┐
│         RAG（检索增强）                                       │
│         （从知识库检索信息，增强回答质量）                     │
└─────────────────────────────────────────────────────────────┘
                              ↑
┌─────────────────────────────────────────────────────────────┐
│         LLM（大语言模型）                                     │
│         （基础推理、理解、生成能力）                           │
└─────────────────────────────────────────────────────────────┘
```

### 功能对比表

| 特性 | RAG | Function Call | MCP | Skill |
|:---|:---:|:---:|:---:|:---:|
| **主要用途** | 知识检索 | 工具调用 | 标准化协议 | 任务组合 |
| **输出** | 增强的回答 | 执行结果 | 集成能力 | 完整任务 |
| **复杂度** | 中等 | 中等 | 高 | 高 |
| **可扩展性** | 有限 | 中等 | 高 | 高 |
| **典型应用** | 知识库问答 | API 调用 | 多工具集成 | 端到端任务 |

## 应用示例

### 示例 1：NL2SQL（自然语言转 SQL）

**问题**：如何用自然语言查询数据库？

**实现方式**：
1. **RAG**：检索 SQL 示例、表结构文档
2. **Function Call**：调用执行 SQL 的函数
3. **组合 Skill**：自然语言 → SQL → 执行 → 结果展示

**工作流程**：
```
用户问："上个月销售额最高的产品是什么？"
     ↓
[理解需求] LLM 分析问题
     ↓
[RAG 检索] 获取表结构、SQL 示例
     ↓
[Function Call] 生成并调用 SQL 执行函数
     ↓
[Skill 整合] 整理结果，返回易懂的回答
```

### 示例 2：智能数据分析 Agent

**功能**：上传 Excel → 分析数据 → 生成图表 → 输出报告

**技术栈**：
| 环节 | 技术 |
|:---|:---|
| **读取文件** | Function Call / MCP |
| **理解数据** | LLM |
| **分析方法** | RAG（检索最佳实践） |
| **生成代码** | LLM |
| **执行代码** | Function Call |
| **完整流程** | Skill 编排 |
