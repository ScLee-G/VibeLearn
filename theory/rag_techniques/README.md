# RAG 技术体系

## 一、RAG 定义

RAG（Retrieval-Augmented Generation）是一种结合检索和生成的技术，让 LLM 能够基于外部知识库生成回答。

## 二、RAG 工作流程

```
用户提问 → 检索相关文档 → 构建提示词 → LLM 生成回答
```

## 三、核心组件

### 1. 文档处理
- 文档加载
- 文本分割
- 向量化

### 2. 向量数据库
- 存储向量嵌入
- 相似度检索
- 增量更新

### 3. 检索策略
- 语义检索
- 混合检索
- 多模态检索

### 4. 提示词构建
- 上下文注入
- 指令设计
- 格式控制

## 四、主流向量数据库

| 数据库 | 特点 | 适用场景 |
|:---|:---|:---|
| Pinecone | 托管服务、易用 | 快速原型 |
| Chroma | 轻量、开源 | 本地开发 |
| Milvus | 高性能、可扩展 | 企业级 |
| FAISS | 高效、Facebook | 研究场景 |

## 五、RAG 优化策略

1. **Chunk 优化**：合理的文本分割大小
2. **检索增强**：多级检索、重排序
3. **提示词优化**：上下文管理、指令工程
4. **评估体系**：BLEU、ROUGE、人工评估

## 六、应用场景

- 知识库问答
- 文档摘要
- 代码检索
- 多模态检索
