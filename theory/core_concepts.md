## RAG（Retrieval-Augmented Generation）

### 什么是 RAG？
**RAG** = **检索增强生成**，是一种结合了信息检索和文本生成的技术，让大语言模型（LLM）能够基于外部知识库来回答问题。

### RAG 的工作流程
```
用户提问 → 检索相关文档 → 构建提示词 → LLM 生成回答
```

### RAG 的核心组件

| 组件 | 说明 |
|:---|:---|
| **文档处理** | 将文档切分成块（Chunks） |
| **向量化** | 将文本转换为向量嵌入（Embedding） |
| **向量数据库** | 存储向量，支持相似度检索 |
| **检索策略** | 语义检索、混合检索等 |
| **提示词构建** | 将检索到的上下文注入提示词 |

### RAG 的优势
- ✅ 减少幻觉（Hallucination）
- ✅ 支持最新信息
- ✅ 可追溯信息来源
- ✅ 保护知识产权

## Function Call（函数调用）

### 什么是 Function Call？
**Function Call** 是 LLM 调用外部工具/函数的能力，让 AI 不是只说话，而是能「做事」。

### Function Call 的工作流程
```
用户请求 → LLM 分析需求 → 判断需要调用工具 → 调用 Function → 获取结果 → 整合回答
```

### Function Call 的典型应用
| 应用场景 | 示例 |
|:---|:---|
| **获取实时数据** | 天气查询、股票行情 |
| **外部 API 调用** | 地图、快递、支付 |
| **代码执行** | 运行 Python 代码、SQL 查询 |
| **系统操作** | 文件读写、数据库操作 |

### Function Call 示例
```python
functions = [
    {
        "name": "get_weather",
        "description": "获取指定城市的天气",
        "parameters": {
            "type": "object",
            "properties": {
                "city": {"type": "string", "description": "城市名"},
            },
            "required": ["city"]
        }
    }
]
```

## MCP（Model Context Protocol）

### 什么是 MCP？
**MCP** = **模型上下文协议**，是一套让 AI 能够与外部资源（文件系统、数据库、API）交互的标准化协议。

### MCP 的核心理念
- **标准化接口**：统一的工具调用方式
- **可扩展**：易于添加新的工具和数据源
- **安全隔离**：沙箱环境，安全可控
- **双向通信**：AI 和工具之间可以持续交互

### MCP 的典型组件
| 组件 | 说明 |
|:---|:---|
| **Servers** | 提供工具和资源的服务端 |
| **Tools** | 可调用的具体功能 |
| **Resources** | AI 可访问的数据源 |
| **Prompts** | 预定义的提示词模板 |

## Skill（技能）

### 什么是 Skill？
**Skill** 是指 AI 能够完成的特定任务能力，通常是将多个 Function 或工作流组合封装而成。

### Skill 的特点
- 🎯 **面向任务**：解决具体问题
- 🔧 **可组合**：多个 Skill 可以组合使用
- 📦 **可复用**：一次编写，多次使用
- 🤝 **与人协作**：可以和人类协作完成任务

### Skill 示例
| Skill 名称 | 功能描述 |
|:---|:---|
| **写代码** | 生成、调试、运行代码 |
| **数据分析** | 读取数据、分析、可视化 |
| **文档处理** | 阅读、总结、翻译文档 |
| **智能助手** | 日程管理、提醒、搜索 |

## 概念对比

### 从「能力层级」看

```
┌─────────────────────────────────────────────────────────────┐
│                         Skill（技能）                         │
│         （面向任务，组合多个工具完成复杂工作）                 │
└─────────────────────────────────────────────────────────────┘
                              ↑
┌─────────────────────────────────────────────────────────────┐
│    Function Call / MCP（工具调用）                           │
│         （调用单个工具，获取外部信息或执行操作）               │
└─────────────────────────────────────────────────────────────┘
                              ↑
┌─────────────────────────────────────────────────────────────┐
│         RAG（检索增强）                                       │
│         （从知识库检索信息，增强回答质量）                     │
└─────────────────────────────────────────────────────────────┘
                              ↑
┌─────────────────────────────────────────────────────────────┐
│         LLM（大语言模型）                                     │
│         （基础推理、理解、生成能力）                           │
└─────────────────────────────────────────────────────────────┘
```

### 功能对比表

| 特性 | RAG | Function Call | MCP | Skill |
|:---|:---:|:---:|:---:|:---:|
| **主要用途** | 知识检索 | 工具调用 | 标准化协议 | 任务组合 |
| **输出** | 增强的回答 | 执行结果 | 集成能力 | 完整任务 |
| **复杂度** | 中等 | 中等 | 高 | 高 |
| **可扩展性** | 有限 | 中等 | 高 | 高 |
| **典型应用** | 知识库问答 | API 调用 | 多工具集成 | 端到端任务 |

## 应用示例

### 示例 1：NL2SQL（自然语言转 SQL）

**问题**：如何用自然语言查询数据库？

**实现方式**：
1. **RAG**：检索 SQL 示例、表结构文档
2. **Function Call**：调用执行 SQL 的函数
3. **组合 Skill**：自然语言 → SQL → 执行 → 结果展示

**工作流程**：
```
用户问："上个月销售额最高的产品是什么？"
     ↓
[理解需求] LLM 分析问题
     ↓
[RAG 检索] 获取表结构、SQL 示例
     ↓
[Function Call] 生成并调用 SQL 执行函数
     ↓
[Skill 整合] 整理结果，返回易懂的回答
```

### 示例 2：智能数据分析 Agent

**功能**：上传 Excel → 分析数据 → 生成图表 → 输出报告

**技术栈**：
| 环节 | 技术 |
|:---|:---|
| **读取文件** | Function Call / MCP |
| **理解数据** | LLM |
| **分析方法** | RAG（检索最佳实践） |
| **生成代码** | LLM |
| **执行代码** | Function Call |
| **完整流程** | Skill 编排 |
